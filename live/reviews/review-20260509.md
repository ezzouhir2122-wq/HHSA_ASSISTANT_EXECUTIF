### À AMÉLIORER

- **equipment/generate_pdf_report.py:39-46** — `subprocess.check_call` pour auto-installer `fpdf2` est un anti-pattern en production. Il installe silencieusement un package dans l'environnement Python de l'utilisateur sans consentement explicite. Mieux : échouer avec un message clair (`pip install fpdf2`) et laisser l'utilisateur décider.

- **equipment/generate_pdf_report.py:50-63** — `safe()` fait une double conversion : d'abord un remplacement manuel des caractères accentués (lignes 59-61), puis un `encode("latin-1")` qui referait le travail. Les remplacements `"é": "\xe9"` etc. sont redondants car `\xe9` *est* le code latin-1 de `é` — autant juste laisser l'encode gérer ça. Simplifier : retirer les 3 lignes de remplacement d'accents et laisser `encode("latin-1", errors="replace")` traiter tous les caractères non-supportés uniformément.

- **equipment/generate_pdf_report.py:94-115** — `ReportPDF` est définie à l'intérieur de `generate_report()`, ce qui la recrée à chaque appel. Pas un bug, mais si le script venait à être appelé en boucle (ex. génération batch), c'est inutilement coûteux. Déplacer la classe au niveau module.

- **.claude/skills/research-agentic-trends/SKILL.md:114-121** — La commande Bash de génération PDF utilise des chemins relatifs (`equipment/generate_pdf_report.py`, `live/research/...`). Si le CWD n'est pas la racine du projet au moment de l'appel, le script échoue silencieusement ou produit une erreur de chemin. Préférer un chemin absolu ou documenter que la commande doit être exécutée depuis la racine du projet.

- **.claude/agents/code-reviewer.md:16** — Le format du nom de fichier de rapport est `live/reviews/review-[YYYYMMDD-HHMM].md` (avec heure), mais l'instruction donnée dans la session courante utilise `review-20260509.md` (sans heure). Incohérence entre l'agent et son déclencheur — aligner les deux sur un seul format.

- **CLAUDE.md:76** — L'entrée `code-reviewer *(agent)*` dans la table "Skills Actifs" utilise une notation informelle `*(agent)*` non définie ailleurs dans le fichier. Les autres skills n'ont pas de qualificatif. Soit documenter la distinction agent/skill, soit uniformiser la notation.

- **live/research/findings-20260509.json:34** — Le champ `source` de la découverte #5 référence deux sources distinctes avec une seule URL (`Ampcome + arXiv — https://...`). Si le JSON est relu programmatiquement pour construire des citations, cette concaténation cassera le parsing. Utiliser un tableau `"sources": [...]` pour les multi-sources.
